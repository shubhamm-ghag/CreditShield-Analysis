# CreditShield-Dashboard

1. ( HEADLINE )
CreditShield Loan Default Risk Analysis = An end-to-end fintech analytics project using Excel, SQL (PostgreSQL), and Power BI to identify high-risk borrowers and optimize lending decisions.

3. ( PURPOSE )
= CreditShield is a Loan Default Risk Analysis Dashboard designed to help fintech companies minimize financial loss.
= By analyzing 1,000 combined records of customer demographics and loan transactions, this project identifies critical default patterns and borrower risk levels.

4. ( TECH STACK )
* Excel: Performed initial data audit and preliminary cleaning (NULL counts, casing, and duplicate removal).
* SQL (PostgreSQL): Conducted deep cleaning and complex analysis using JOINs and Window Functions.
* Power BI: Developed a multi-page interactive dashboard with custom professional branding.
  
5. ( Data Source )
Source = Google. 

6. ( BUSINESS INSIDE )
* Default Risk: Discovered a 17% default rate, meaning approximately 1 in 6 customers fail to repay their loans.
* High-Risk Segments: Identified that customers with a credit score below 600 represent the highest risk group.
* Employment Trends: Found that freelancers have a significantly higher default rate compared to salaried employees.
